# 演示指南

> 供演示准备使用，涵盖系统亮点、演示流程、预期输出。

## 系统亮点（Demo 可讲）

1. **离线可用** — 断网也能跑，传统算法产出 393 条结构化差异
2. **多模型池** — 10+ 模型自动切换，一个挂了换下一个
3. **反幻觉** — L2 字符串验证 + L3 LLM 交叉校验，两层把关
4. **自进化** — 每跑一次学一次，下次提示词自动带上历史经验
5. **严谨模式** — 独立交叉验证，准确率更高
6. **实时流式** — SSE 推送进度，不是轮询

## Demo 流程 (5-7 分钟)

### 第一幕：上传 (1 min)

1. 打开 `http://localhost:8000`
2. 展示拖拽上传（拖一个 PDF 进去，全页 overlay 动画）
3. 勾选"严谨模式"
4. 点"开始智能比对"

### 第二幕：实时进度 (1-2 min)

1. SVG 环形进度条分阶段动画
2. 差异逐条实时弹出（不等到结束才显示）
3. 右上角实时计数：已耗时 / 变化数 / 高风险数

### 第三幕：结果页 (2 min)

1. 风险筛选（高风险 / 中风险 / 低风险）、类型筛选
2. 右侧悬浮导航点阵 — 每个点代表一条差异，颜色区分风险等级和确认状态
3. 点击导航点 → 平滑滚动到对应卡片
4. 备注框：输入文字 → 失焦自动保存（无保存按钮）
5. 下载 PDF 报告

### 第四幕：自进化 (1 min)

1. 回到首页 → 点击 header 中"进化记录"
2. 展示累计运行次数、平均差异数、高频风险类别
3. 排序/筛选（按日期/差异数/高风险，全量/LLM/离线）

## 环境准备

```bash
# 1. 启动服务
make web

# 2. 如果有 API Key，可以演示完整 LLM 管线
#    如果没有，离线模式也完全可以演示（只是没有风险分类文案）
cp .env.example .env
# 编辑 .env 填入 LLM_API_KEY

# 3. 测试文件在 docs/ 目录
#    docs/天猫服务协议2015(2).pdf  (V1, 旧版)
#    docs/天猫服务协议2026(2).pdf  (V2, 新版)
```

## CLI 备用演示

如果 Web 演示出问题，CLI 命令行也能展示完整流程：

```bash
# 离线模式（无需 API Key）
python -m src.main docs/天猫服务协议2015\(2\).pdf docs/天猫服务协议2026\(2\).pdf --offline -o /tmp/result.json

# 输出 JSON 报告
cat /tmp/result.json | python -m json.tool | head -40
```
