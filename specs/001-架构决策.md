# 架构决策记录

## AD-001: LLM 调用策略
**决策**：先整份合同一次调用，遇到上下文窗口溢出再切分。
**已知坑位**（待验证）：
- 长合同+输出结果可能超出上下文窗口
- 一次调用处理两份合同，token 消耗线性增长
- 表格密集区域可能占大量 token 但语义密度低
**回退方案**：按章节切分，逐章比对，最后汇总

## AD-002: 输出格式
**决策**：结构化 JSON，字段包含差异列表、风险分类、提示文本
**理由**：便于后续接 Web UI、结构化存储、自动化处理

## AD-003: 风险分类体系
**决策**：动态增长型分类库
- 初始种子：我根据珠宝行业背景拟定 ~10 个常见分类
- 运行时：每次比对遇到新风险类型，自动入库
- 频次统计：每个分类记录命中次数，高频分类标注为重点关注
- 二次校验：频次超过阈值的类型，触发更细颗粒度的检查提示
